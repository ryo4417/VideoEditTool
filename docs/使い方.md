# VideoEditTool 使い方（試作機ガイド）

動画から「無音・口ぐせ・言い直し・繰り返し」を削って、テンポよく仕上げるツールです。
**まずは「1. GUIで使う」だけ読めば始められます**（3ステップ）。他は必要になったら読めばOK。
編集はルールで行い、AIは補助（品質チェック）です。

---

## 0. 準備（初回のみ）

必要なもの:
- **Python 3.11+**
- **ffmpeg / ffprobe**（PATH に通す。例: `winget install Gyan.FFmpeg`）

あると便利（任意・無くても無音ベースの編集は動きます）:
- **faster-whisper** … 文字起こし（フィラー/重複/言い直しに必要）
- **yt-dlp** … YouTube等のURLから動画を取り込む機能に必要
- **Ollama** … AIによる品質採点（7章）

セットアップ＆GUI起動（推奨・ワンコマンド）:
```powershell
cd ~\Desktop\VideoEditTool
.\start.ps1                 # 依存を入れてGUIを起動（ブラウザが開く）
.\start.ps1 -WithWhisper    # フィラー/重複/言い直しを使うなら（ローカルWhisper導入）
.\start.ps1 -Sample         # 試用サンプル動画 sample.mp4 も生成
```

手動で試すなら:
```powershell
python -m pip install PyYAML          # 必須
python -m pip install faster-whisper  # 文字起こしを使う場合のみ（重い）
python -m pip install yt-dlp          # URL取り込みを使う場合のみ
python scripts\make_sample.py         # 試用サンプル sample.mp4 を作る（任意）
```

## 1. GUIで使う（いちばん簡単・おすすめ）

### 起動
- **`start.bat` をダブルクリック**（いちばん簡単。ブラウザで編集画面が開きます）
- またはPowerShellから:
```powershell
powershell -ExecutionPolicy Bypass -File .\start.ps1
```
（うまく開かないときは `python -m ui.web.server` でもOK）

### たった3ステップ

**① 動画を入れる**
入れ方はどれでもOK。**複数まとめて入れてもOK**です:
- 画面の点線ワクに動画を**ドラッグ&ドロップ**（複数まとめてドロップ可）
- 「**ファイルを選択**」ボタンで選ぶ（複数選択可）
- パス入力欄に動画のパスを直接入力（前後の `"` は自動で取り除かれます）
- 「**YouTube等のURLから取り込み**」欄にURLを貼って「URLから取り込み」（yt-dlp が必要）
- 手元に動画が無ければ「**お試し動画を作る**」ボタンで練習用の動画が用意されます

複数入れると下に**動画リスト**が出ます。クリックで切り替えでき、**動画ごとに解析結果・編集は別々に保持**されます。

**② 何を消すか選んで「解析」**
- チェックボックスから消したいものを選びます（マウスを乗せると説明が出ます）:
  - ☑ **無音** … 長い無音をまるごとカット
  - ☑ **テンポ** … 長い「間」だけ縮めて自然さは残す
  - ☑ **フィラー** … 「えー・あの・um」などの口ぐせ
  - ☑ **重複** … 同じ言い回しの繰り返し
  - ☑ **言い直し** … 言い直した前半
- 「フィラー・重複・言い直し」を選ぶと自動で音声の文字起こしを使います（初回だけ準備に数分）。
- **言語**（自動/日本語/英語）と **精度**（tiny=最速 〜 large-v3=最高精度）を選べます。精度が高いほど正確ですが遅く、初回のモデル取得も大きくなります。
- **「解析」** を押すと処理中は画面にオーバーレイと**経過秒数**が出て、その間は操作がロックされます。
- 終わると動画の下に **緑=残す / 赤=カット** の帯（＋波形）が出ます。

**③ 手で細かく調整する（自動カットの後）**
自動で付いた赤いカット帯を、自分で自由に直せます:
- **移動**: 赤帯をドラッグ
- **伸縮**: 赤帯の左右の端をつまんでドラッグ
- **削除**: 赤帯をダブルクリック（または一覧の ✕）
- **新規追加**: タイムラインの何もない所を横にドラッグ、または「**＋カット追加**」ボタン
- 一覧では開始・終了の秒数を直接入力して微調整・ON/OFF・削除ができます。各カットに対応する文字起こしも表示されます
- **やり直しても安心**:
  - 「**↩ 元に戻す**」で直前の編集を取り消せます（Undo）
  - 「**自動カットに戻す**」で自動検出の状態に戻せます
  - 波形の下に「**自動検出（元）**」の帯が別に出るので、今の編集と元の自動結果を見比べられます
- **波形の見やすさ**: 波形は右上の **＋/−でズーム**（拡大）でき、横スクロールで送れます
- **文字起こしトラック**: 波形の下に文字起こしが時間軸で並びます。カットに掛かる語は**取り消し線**、語を**クリックするとその位置に頭出し**できます
- 「**編集後プレビュー**」にチェックして再生すると、**カット後の状態で再生**され仕上がりを耳で確認できます

**④ 書き出し**
- よければ形式を選んで **「書き出し」**。「実カット動画も出力」で編集済み mp4 ができます。
- 保存先は既定で `output/`。**出力先フォルダ**を指定でき、**Googleドライブ**に出したいときは下の 5.5 参照。

> 困ったとき: 何もカットされない → ②で項目にチェックが入っているか確認。フィラーが残る → 話し方によっては文字起こしが「えー」を拾えないことがあります。

## 2. CLIで使う

```powershell
# 解析→書き出し（既定 json のカットリスト）
python -m ui.cli 入力.mp4

# 案件プロファイル（config/profiles/*.yaml）
python -m ui.cli 入力.mp4 --profile youtube
python -m ui.cli 入力.mp4 --profile interview

# 書き出し形式
python -m ui.cli 入力.mp4 --format html      # 編集確認プレビュー(HTML)
python -m ui.cli 入力.mp4 --format edl       # EDL
python -m ui.cli 入力.mp4 --format fcpxml    # NLE連携(FCPXML)

# 品質レポート / 実カット動画
python -m ui.cli 入力.mp4 --report
python -m ui.cli 入力.mp4 --render

# 複数形式をまとめて（解析は1回だけ＝速い）
python -m ui.cli 入力.mp4 --format fcpxml,edl,json --report

# 複数ファイルをまとめて処理（ワイルドカード可・モデルを使い回す）
python -m ui.cli *.mp4 --format edl --output-dir out

# 文字起こし本文を確認 / 出力先
python -m ui.cli 入力.mp4 --transcript
python -m ui.cli 入力.mp4 --output-dir out
```

## 3. 編集ルール（何をカットするか）

すべて `config/default.yaml` の `rules:` で ON/OFF・調整できます（コード変更不要）。
案件ごとに `config/profiles/*.yaml` で上書き。

| ルール | 内容 | 必要な解析 | 主な設定 |
|--------|------|-----------|----------|
| `silence` | 一定以上の無音をカット | 無音検出（ffmpeg） | `min_cut_sec`, `keep_padding_sec` |
| `filler` | フィラー語（えー/あの/um…）を除去 | 文字起こし | `words`（対象語リスト）, `padding_sec` |
| `duplicate` | 連続する繰り返し語句を除去 | 文字起こし | `min_words`, `max_words` |
| `restate` | 言い直し（隣接する類似句）を除去 | 文字起こし | `min_words`, `max_words`, `sim_threshold` |
| `tempo` | 長い間の超過分だけカットし自然な間を残す | 無音検出 | `target_pause_sec`, `min_gap_sec` |

例: フィラー検出を有効化して CLI 実行
```yaml
# config/default.yaml
analysis:
  transcript: { enabled: true, model: base, language: ja }
rules:
  filler: { enabled: true }
```
```powershell
python -m ui.cli 入力.mp4 --format html --report
```

## 4. 文字起こし（ローカルWhisper）

- フィラー/重複/言い直しは、発話内容の**文字起こし**が前提です。
- ローカルの `faster-whisper` を使用（オフライン。初回にモデルを自動DL）。
- GUIでは解析パネルの「精度」で、CLI/設定では `config/default.yaml` の `analysis.transcript` で選べます:
  - `model`: `tiny`/`base`/`small`/`medium`/`large-v3`（大きいほど高精度・低速）
  - `language`: `ja`/`en`（自動判定は `null`）
- AIが無くても**無音ベースの編集は常に可能**です（文字起こしは補助）。

## 5. 書き出し形式

| 形式 | 用途 |
|------|------|
| `json` | カットリスト（他ツール連携・確認） |
| `html` | 人間が見て確認するプレビュー（タイムライン・警告・候補表） |
| `edl` | 汎用の編集リスト（映像＋音声） |
| `fcpxml` | NLE（Final Cut 系）へ読み込む用 |
| 実カット動画 | `--render`。keep区間だけを連結した mp4 |

## 5.5 Googleドライブで入出力する

「Google Drive for Desktop（ドライブ デスクトップ版）」を使うと、Driveがパソコンの
フォルダ（例 `G:\マイドライブ`）として見えるので、そのまま入力・出力に使えます。

1. [Google Drive for Desktop](https://www.google.com/intl/ja/drive/download/) を入れてサインイン
2. **入力**: Drive上の動画をGUIにドラッグ、またはパス欄に `G:\マイドライブ\...\動画.mp4` を入力
3. **出力**: 書き出しパネルの「出力先フォルダ」に Drive のフォルダ（例 `G:\マイドライブ\編集済み`）を指定
   - Driveを検出すると「**Googleドライブに保存**」ボタンが出て、ワンタップで出力先が埋まります
   - CLI なら `--output-dir "G:\マイドライブ\編集済み"`

出力したファイルは Drive が自動でクラウドへ同期します。追加設定やAPIキーは不要です。

## 6. 設定のしくみ

- すべての編集条件は `config/*.yaml`。コードは触りません。
- `config/default.yaml` … 既定値
- `config/profiles/<名前>.yaml` … 案件プロファイル（default を上書き）
- GUIのプロファイル選択（youtube / interview）では、選ぶと画面に内容の説明が表示されます。
- 起動時にスキーマ検証が走り、未知キー・型ミス・不正な形式はエラーで教えてくれます。

## 7. AI補助の品質採点（ローカルLLM / Ollama・任意）

- 品質レポートに、ローカルLLMによる**採点(0-100)・改善案・編集漏れの指摘**を付与できます。
- **Ollama** を使用（オフライン）。準備:
  1. Ollama をインストールし起動（`http://localhost:11434`）
  2. モデルを用意（例: `ollama pull llama3.1`）
  3. `config/default.yaml` の `ai.enabled: true`（`provider: ollama`, `model: llama3.1`）
- AIは**補助**です。無効でも、また Ollama に繋がらなくても、編集は成立します（失敗時は警告に記録）。
