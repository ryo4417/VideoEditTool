# VideoEditTool コードレビュー依頼（GPT向け・自己完結）

このドキュメント単体でレビューできるよう、目的・設計・現状・既知の懸念・レビュー観点を

## 1. 目的・思想
汎用の「動画カット編集支援ツール」。編集は**独自アルゴリズム（ルール）**で行い、
AIは補助（品質採点・改善案）に限定。**オフライン完結**・**拡張性最優先**・**config駆動**・
**モジュール間依存最小**・**UIと処理の分離**が設計原則。

## 2. 技術スタック
- Python 3.11 / 標準ライブラリ中心。外部依存は **PyYAML** のみ必須。
- 動画処理は **ffmpeg/ffprobe**（subprocess）。
- 文字起こし: **faster-whisper**（ローカル・任意）。AI採点: **Ollama**（ローカルLLM・任意）。
- GUI: 標準ライブラリの http.server + 素のHTML/JS（追加依存なし）。
- テスト: pytest（77件）。

## 3. アーキテクチャ / データフロー
```
入力 → 解析(analyzers) → 編集候補(rules) → タイムライン整形 → 品質チェック(+AI任意) → 書き出し
       silence/speech/transcript  silence/tempo/filler/     keep区間導出      QualityChecker    json/edl/
                                   duplicate/restate                                            html/fcpxml/実カット
```
- `core/`（純粋な共通層。他に依存しない）: models（TimeRange/AnalysisResult/EditCandidate/Timeline）、
  config（YAML+深いマージ+スキーマ検証）、ffmpeg（probe/silencedetect/waveform/render）、
  registry（アナライザ・ルールの名前登録）、analyzer/rule 基底。
- `audio/`（silence, speech, transcribe/transcript）、`rules/`（silence, tempo, filler, duplicate, restate + engine）、
  `timeline/`（keep区間の補集合・近接マージ・短区間除去）、`export/`、`quality/`（checker + ai_assist）。
- `pipeline.py`（配線層。coreに依存させない）: `analyze()` / `recompute(enabled_indices)` / `export_result()` / `run()`。
- `ui/cli.py`（CLI）、`ui/web/`（ローカルWeb GUI）。

## 4. 拡張のしくみ（重要）
- アナライザ/ルールは `@ANALYZERS.register("名")` / `@RULES.register("名")` で登録するだけで追加可能。
- 有効化・閾値は全て `config/*.yaml`。`analysis` と `rules` は「開放セクション」（プラグイン名は任意）。
- 新ルール追加は「ファイル追加 + config追加 + builtin import 1行 + テスト」の4手順。エンジン改変不要。

## 5. 主要ロジック抜粋

### タイムライン（カットの補集合＝残す区間）
```python
def complement_ranges(ranges, total, start=0.0):
    ordered = sorted(ranges, key=lambda r: r.start)
    segments, cursor = [], start
    for r in ordered:
        s = max(r.start, start)
        if s > cursor: segments.append(TimeRange(cursor, s))
        cursor = max(cursor, min(r.end, total))
    if cursor < total: segments.append(TimeRange(cursor, total))
    return segments
```

### ルールエンジン（config で enabled のものだけ実行）
```python
for name, options in rules_config.items():
    if not options.get("enabled", False): continue
    if name not in RULES.names(): raise KeyError(...)   # 未登録は気づけるように
    self._rules.append(RULES.create(name, options))
```

### フィラー除去（内容ベース・正規化一致）
```python
fillers = {_normalize(f) for f in self.opt("words", DEFAULT_FILLERS)}
for w in words:
    if _normalize(w.text) in fillers:
        candidates.append(EditCandidate(TimeRange(w.start-pad, w.end+pad), CUT, "filler", ...))
```

### ffmpeg 無音検出（stderrをパース）/ probe（失敗は MediaProbeError）
- `silencedetect=noise={db}dB:d={sec}` の stderr から silence_start/end を抽出。
- probe は returncode 非0で `MediaProbeError` を送出（CLIは[エラー]表示、GUIは400）。

## 7. 私（実装者）が把握している改善点・懸念 ← ここを重点的に検証してほしい
1. **複数ルールのカット候補が重なる**（silence+tempo+filler 等）。keep区間は補集合で正しく出るが、
   候補一覧・`num_cuts` は重複計上。候補のマージ/優先度/`confidence` 活用が未実装。
2. **silence と tempo は同じ無音を別方針で扱う**ため同時有効化は矛盾。ガード/警告なし。
3. **発話の途中カット（filler/duplicate/restate）でクリック音**の恐れ。境界スナップ/クロスフェード無し。
4. **render_cuts が全keep区間を1つの filter_complex** に展開＋フル再エンコード。長尺で
   ffmpegのグラフ/引数上限や速度・画質劣化の懸念。segment+concat(ストリームコピー)未検討。
5. **GUIはローカルパスを任意に受理**（/media, /api/analyze）。localhost限定・単一ユーザ前提だが
   パストラバーサル/任意ファイル読取・任意 output_dir 書込の余地。認証なし。
6. **Whisperモデルを解析毎に再ロード**（キャッシュ無し）→ GUI連続解析で遅い。
7. **spec §6 の「指定時間以上/以下」ルールが未実装**（timeline.min_segment で短区間除去のみ）。
8. **duplicate の min_words=1** は自然な繰り返し（「そうそう」等）を過剰カットしうる。
9. CI/リンタ未整備（ruff設定はあるが未実行）。GUIのフロントは自動テスト無し。
10. `/media` が要求レンジ全体をメモリ読み。長尺で一時的メモリ増。
