# 動画カット編集支援ツール 仕様書 v0.1

## 1. プロジェクト概要

### 目的

本ツールは、動画のカット編集業務を効率化することを目的とした動画編集支援ツールである。

特定案件専用ではなく、様々な動画編集案件に対応できる汎用ツールとして開発する。

最終目標は、編集者が最小限の確認だけで高品質なカット編集を完了できる環境を構築することである。

---

## 2. 開発コンセプト

本ツールはAIによる編集ツールではない。

編集処理は独自アルゴリズムによって行い、

AIは

* 編集結果の確認
* 品質評価
* 編集漏れ検出
* 改善提案

など補助用途のみに使用する。

編集ロジックは可能な限りオフラインで完結できる構成を目指す。

---

# 3. 開発方針

本開発ではMVPを最優先とする。

ただし、MVPは最低限の機能ではなく、

**最終目標へ到達するための土台となる設計**

を最優先とする。

そのため、

* 後から追加できる
* 後から削除できる
* 後から差し替えられる

設計を採用する。

---

# 4. 設計理念

Cloud Codeは以下を遵守する。

* 保守性を最優先
* モジュール化を徹底
* 共通処理をまとめる
* 後からAI追加可能
* 後からアルゴリズム変更可能
* UIと処理を分離
* 設定変更のみで動作変更可能

コードの短さより拡張性を優先する。

ルールは後から自由に追加可能とする。

例

・無音

・重複

・言い直し

・テンポ

・フィラー

・指定時間以上

・指定時間以下

・その他

現段階では内容を固定しない。

* 数値変更

* 閾値変更

* 編集ルール追加

ができる構成にする。

## 私から追加したい内容

この仕様書には、**「マニュアル学習システム」**を将来構想として入れておくことをおすすめします。

具体的には、「案件ごとの編集マニュアルを読み込み、その内容を編集ルールへ変換する仕組み」です。

例えば、

* 「無音は0.3秒以上カット」
* 「言い直しは削除」
* 「笑い声は残す」
* 「相槌は削除しない」

といったルールを、設定ファイルやルールエンジンに反映できる構造を最初から想定しておくと、この案件だけでなく別のクライアント案件にも対応しやすくなります。

この設計であれば、案件獲得後に仕様が変わっても大幅な作り直しを避けやすく、長く使える編集支援ツールへ発展させやすいと思います。
